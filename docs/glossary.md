# Glossary

| Term | Meaning |
| --- | --- |
| Agent OS | The operating layer around agents: instructions, tools, memory, policy, runtime, deployment, and audit |
| Control plane | The agent or process that coordinates repo changes, tasks, tests, and handoffs |
| Worker profile | A durable named local agent identity, often mapped to a role or lane |
| Gateway | A service that routes chat, mobile, or external messages into agent sessions |
| Harness | A structured runtime for long-running or multi-step agent workflows |
| MCP | Model Context Protocol, used to connect agents to tools and data |
| Skill | A reusable workflow or knowledge package an agent loads when relevant |
| Hook | An enforcement point that can block, log, or modify agent behavior |
| Provenance | The record of why a decision was made, by whom/what, and from which sources |
| Trust tier | A permission class describing what an agent/tool may read, write, or execute |
